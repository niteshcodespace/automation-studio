# Automation Studio Development Logs

> Last updated: 2026-07-13

## Purpose

Development logs provide an evidence-based record of each sprint and story so an engineer can understand what changed, why it changed, what was verified, and what remains after a long gap.

## Format

- Sprint files record objectives, included stories, overall status, outcomes, and remaining work.
- Story files record the objective, branch, implementation and documentation changes, important decisions, verification evidence, problems, Definition of Done, and next action.
- Status is always explicit: **Completed**, **In progress**, **Planned**, or **Blocked**.
- Repository contents and Git history are the primary evidence. Session-reported facts are labeled when retained command output is unavailable.
- Logs are appended or carefully revised; useful history is not erased.
- No log may include passwords, tokens, private credentials, or resolved secret values.
