# AS-017 - Environment Management

> Last updated: 2026-07-24

## Objective

Define and then deliver Project-scoped Environment management for base URLs, non-secret runtime
configuration, external secret references, lifecycle, and default selection while evolving the
existing Environment aggregate and preserving Execution history.

## Current Status

**AS-017A and AS-017B completed; the migration, focused migration tests, and full Maven suite
pass.**

AS-017C through AS-017G remain planned and are not implemented. AS-017B changes only the database
migration, its PostgreSQL migration coverage, and test fixtures that must insert rows into the
expanded schema; it does not claim that the Java model or API implements Environment management.

## Important Decisions

- Evolve the existing aggregate; never create a duplicate Environment model/table.
- Preserve existing identifiers, ownership, names, base URLs, timestamps, uniqueness, and
  restrictive Execution relationships.
- Add description, type, two non-null JSONB objects, default flag, optimistic version, ARCHIVED,
  an ACTIVE-default check constraint, and a partial Project-default unique index in AS-017B.
- Require an absolute HTTP(S) base URL with a host and no user-info or fragment.
- Limit each serialized JSON object to 64 KiB.
- Contractually restrict configuration to non-secret runtime data and recursively reject the
  documented case-insensitive secret-shaped keys as defence in depth, without claiming arbitrary
  JSON can be proven secret-free.
- Accept only nonblank URI-style strings in secretReferences; credential resolution remains
  runner-only, and rejected values/JSON payloads never enter logs or errors.
- Serialize default changes through a Project lock and retain a partial unique index as the final
  invariant safeguard.
- Clear the default when it leaves ACTIVE and never auto-promote a replacement.
- Require `If-Match` expected versions for PUT, both PATCH operations, and DELETE; reject missing
  or malformed headers with 400 and version/JPA conflicts with 409 while JPA owns persisted versions.
- Translate duplicate names, protected deletion, default constraints, and optimistic conflicts to 409.

## Assumptions and Open Gate

No existing convention defines a JSON payload byte limit, so AS-017A establishes and retains
64 KiB per JSON object as the accepted bounded-input rule.

Legacy rows have no authoritative persisted Environment classification. AS-017B therefore
backfills them to TEST explicitly to preserve upgrade compatibility. This is not a claim about
their actual targets; users may correct classifications through the API after delivery.
