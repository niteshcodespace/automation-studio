# AS-005 — AI-native Software Architecture Baseline

> Last updated: 2026-07-13

## Objective

Define an implementable AI-native architecture for Automation Studio that supports an OrangeHRM Playwright smoke demonstration while preserving extensibility, security, auditability, and practical v0.1 delivery.

## Completed Architecture Work

- Defined logical control, execution, AI capability, and integration planes.
- Established a Spring Boot modular-monolith control plane and a separate execution runner.
- Defined independent engine plugins behind a shared, versioned contract.
- Specified durable asynchronous execution using PostgreSQL job claiming and a transactional outbox for v0.1.
- Defined execution states, attempt/lease behavior, normalized results, evidence, artifact metadata, and audit expectations.
- Defined optional, advisory AI boundaries and a governed future MCP adapter.
- Documented v0.1 deployment and enterprise evolution paths.
- Added explicit architecture goals to the system architecture.

## Important Decisions

- The control plane owns authoritative business state.
- Automation runs in the execution plane, never on an API request thread.
- Playwright Java is the first engine but does not become a control-plane dependency.
- PostgreSQL is the initial metadata store, work-claim mechanism, and outbox foundation.
- Artifact bytes remain outside PostgreSQL behind a storage port.
- Secrets remain references until scoped execution-time resolution.
- AI is optional, evidence-grounded, audited, and non-authoritative.
